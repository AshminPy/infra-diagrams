# infra-diagrams

Enterprise-ready architecture diagrams generated by AI agent pipeline.

## How it works

1. Point the diagram agent at a GitHub repo or workflow description
2. Scanner extracts all components (Terraform, K8s, Dockerfiles, configs)
3. Architect agent generates diagrams for all 4 types
4. Validator agent checks completeness, traffic flow, clean layout
5. Human approves → PR opened here with SVG output

## Diagram types

| Type | Description |
|---|---|
| `high-level` | Service-to-service overview, cloud boundaries |
| `low-level` | Component internals, config details |
| `network-flow` | VPC/subnet layout, ingress/egress, security groups |
| `user-flow` | Request path from user → service → data |

## Icon sets

Run `bash scripts/download_icons.sh` to pull the latest official icons.
Then `python3 scripts/build_icon_index.py` to regenerate the lookup table.

## Editing diagrams

All outputs are SVG — open in draw.io, Figma, or any vector editor.

## Environment variables

| Variable | Description |
|---|---|
| `ICON_BASE_PATH` | Absolute path to the `icons/` directory in this repo |
| `GITHUB_TOKEN` | Scoped to this repo only (contents:write, pull-requests:write) |
| `ANTHROPIC_API_KEY` | For Scanner and Validator agents |
